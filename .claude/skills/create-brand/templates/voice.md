## 2. Voice attributes

<!-- ✏️ REPLACE — choose 3-5 attributes. Document each in this exact 4-line shape so it's
testable. "We are NOT" prevents over-correction; the two "sounds like" lines are the gold.
EXAMPLE (Vela):

### Clear over clever
- **We are:** plain, concrete, one idea per sentence.
- **We are not:** dumbed-down or so jargon-free we go vague.
- **Sounds like:** "Async updates replace your daily standup. Write once; everyone reads on their own time."
- **Does NOT sound like:** "Unlock next-gen synergy with our revolutionary collaboration paradigm."

### Calm, not hypey
- **We are:** measured, confident, low on adjectives and exclamation marks.
- **We are not:** flat or boring — calm still has a point of view.
- **Sounds like:** "This takes about five minutes to set up."
- **Does NOT sound like:** "🚀 You won't BELIEVE how fast this is!!!"

### Candid
- **We are:** honest about limits and trade-offs.
- **We are not:** negative or self-deprecating.
- **Sounds like:** "Vela won't replace real-time pairing. It replaces the meeting you have to recap anyway."
- **Does NOT sound like:** "Vela does everything, for everyone, perfectly."
-->

## 3. Tone spectrum

The voice (§2) stays fixed; tone dials it up or down by context.

### By channel

<!-- ✏️ REPLACE — adapt rows to the brand's real channels.
EXAMPLE (Vela):

| Channel        | Tone                                   | Mini-example |
|----------------|----------------------------------------|--------------|
| Blog           | Informative, conversational, useful    | "Let's walk through how async updates actually save time." |
| LinkedIn       | Professional, thought-provoking, concise | "Three things we learned shipping 50 async releases." |
| X / short-form | Punchy, direct, occasionally dry        | "Your standup is a meeting to recap work nobody read. Write it down." |
| Email          | Personal, helpful, action-oriented      | "We built something we think your team will actually use." |
| In-app / UI    | Brief, clear, reassuring                | "Saved. Your update is live for the team." |
| Docs           | Patient, step-by-step, precise          | "If you see this error, here's how to fix it." |
| Release notes  | Factual, benefit-first, no hype         | "You can now schedule updates. Write Friday's recap on Thursday." |
-->

### By situation

<!-- ✏️ REPLACE.
EXAMPLE (Vela):

| Situation              | Tone shift                                  |
|------------------------|---------------------------------------------|
| Product launch         | Dial up confidence; still no hype words.    |
| Incident / outage      | Dial up warmth and accountability; transparent. |
| Price change / bad news| Honest, respectful, solution-oriented.      |
| Customer story         | Celebratory, specific, credit the customer. |

**Rule:** the attributes never disappear — context only shifts the balance. A "calm + candid"
brand dials up candor in an incident and calm in a launch.
-->

## 4. Style rules

<!-- ✏️ REPLACE — make every mechanical choice once so output is consistent. Pick a side per row.
EXAMPLE (Vela):

| Rule                     | Choice                                   |
|--------------------------|------------------------------------------|
| Oxford comma             | Yes — "fast, clear, and calm"            |
| Headings                 | Sentence case — "How to get started"     |
| Contractions             | Use them — "we're", "it's"               |
| Em dash                  | Spaced — "this — and more"               |
| Numbers                  | Spell out 1-9, numerals 10+              |
| Percentages              | Use the % symbol — "40%"                 |
| Dates                    | "16 Jun 2026"                            |
| Exclamation marks        | Rare; never more than one                |
| ALL CAPS                 | Never; use **bold** for emphasis         |
| Emoji                    | Minimal; none in product/docs, sparing on social |
| Links                    | Descriptive text, never "click here"     |
-->
