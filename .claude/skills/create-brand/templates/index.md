# Brand kit — NAME

The portable source of truth for the NAME brand. Anything that produces content for NAME — a
Claude Code skill, a `CLAUDE.md`, or a person — should read this file and the files it points to.
The kit is plain Markdown + assets; it does not need to be a skill to work.

## What's in this kit

| File                | What it covers |
|---------------------|----------------|
| `strategy.md`       | Why the brand exists, who it serves, messaging pillars |
| `voice.md`          | Personality, voice attributes, tone, style rules, terminology |
| `visual.md`         | Logo, colour palette (HEX), typography, imagery |
| `examples.md`       | Gold-standard pieces + anti-examples |
| `assets/README.md`  | Index of logos / icon / fonts with public URLs (Google Drive) |

## How to use it (read in this order)

1. **`strategy.md`** — decide which messaging pillar the piece should lead with.
2. **`voice.md`** — how to write it (personality, attributes, tone, terminology).
3. **`visual.md`** — how it should look (exact HEX, fonts, the right logo variant).
4. **`examples.md`** — match these; when in doubt, imitate the gold-standard samples.
5. **`assets/README.md`** — pull the right asset by its public URL.

## Self-check before shipping

- Does it lead with a real messaging pillar (`strategy.md`)?
- Does it pass the voice attributes' "does NOT sound like" tests (`voice.md`)?
- Preferred terminology used, banned terms avoided (`voice.md`)?
- For visual artifacts: correct logo variant, palette, and fonts (`visual.md`)?
- Would it sit comfortably next to the on-brand samples (`examples.md`)?
